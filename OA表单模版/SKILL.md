# OA表单模版

## 何时使用

当用户给出参考表单、截图、HTML、Word、Excel或流程说明，希望生成：

- 与现有OA风格一致的HTML可交互原型
- 报价流程表单
- 项目立项表单
- 采购/风控/审批单
- 成本测算与明细录入表单
- 适合桌面直接打开预览的正式表单页面

本技能优先服务“企业正式审批单/流转单”风格，而不是营销型页面或通用后台表单。

## 目标输出

默认输出到桌面或用户指定路径的单文件HTML原型，要求：

- 可直接浏览器打开
- 可交互输入
- 风格与既有OA表单统一
- 结构完整、字段清晰
- 在PC端优先，移动端兼容查看

## 固定风格

### 1. 页面气质

统一采用“白底正式审批单”风格：

- 白色主页面
- 浅蓝色标签底和分区底
- 浅蓝细边框
- 深灰正文
- 红色必填星号
- 居中主标题

禁止做成卡片后台风、深色风、营销落地页风。

### 2. 设计令牌

默认使用以下颜色和视觉参数：

- 线框色：`#9fc3e7`
- 加强线框：`#6fa7dd`
- 标签底：`#dfeefa`
- 分区底：`#e9f3fc`
- 页面底：`#f5f8fb`
- 正文字色：`#222`
- 标题色：`#111`
- 辅助文字：`#6a7a89`
- 只读高亮：`#1d3f6c`
- 必填红色：`#ff0000`

### 3. 字号体系

- 主标题：`20px`
- 主表正文：`14px`
- 子区标题：`13px`
- 说明文字：`12px`
- 公式/弱提示：`11px`

除非参考文件明显不同，否则不要偏离这套字号体系。

### 4. 布局尺寸

- 页面容器宽：`min(1160px, calc(100vw - 24px))`
- 主表宽：`min(1120px, 100%)`
- 页面上边距：`20px`
- 页面底部留白：`28px`
- 表格单元格内边距：`6px 8px`

## 固定结构

默认按以下结构组织OA表单：

### 1. 页头区

包含：

- 流程名称/单据名称
- 单号/文号
- 申请单位/公司/申请部门等元信息

标题居中，元信息放顶部行内或右上角。

### 2. 基本信息区

优先放：

- 项目名称
- 客户名称/申请对象
- 项目类别/类型
- 部署方式/申请类型
- 项目阶段
- 开始/结束时间
- 预算/金额

这部分适合2列、4列或6列的表格混排。

### 3. 业务说明区

用于长文本字段：

- 背景说明
- 建设内容
- 需求描述
- 范围说明
- 边界说明
- 跟踪情况
- 风险说明

默认用整行大文本框承载。

### 4. 明细测算区

用于：

- 报价明细
- 成本测算
- 毛利测算
- 研发投入
- 交付投入
- 第三方采购

必须支持明细行、汇总值、公式联动。

### 5. 协同对象区

用于：

- 客户联系人
- 审批对象
- 第三方供应商
- 采购明细

默认做成可增删改复制的动态子表。

### 6. 附件与收尾区

用于：

- 风险/承诺
- 是否用印
- 附件上传
- 经办人
- 申请日期
- 申请部门
- 申请公司

## 交互规则

### 1. 输入控件

统一采用弱控件感：

- 输入框默认无边框
- 背景透明
- 字体继承表格正文
- 聚焦时轻微高亮

这类表单不是传统“输入框很多的后台表单”，而是“内容嵌在单元格里的审批单”。

### 2. 动态子表

凡是以下场景，优先支持增删改：

- 联系人
- 明细行
- 成本项
- 采购项
- 附件项

默认提供：

- 新增
- 删除
- 复制

按钮样式小而克制，不做大按钮工具栏。

### 3. 自动计算

涉及以下字段时默认考虑联动：

- 数量 * 单价
- 人天 * 单价
- 成本汇总
- 毛利测算
- 采购总额
- 驻场成本

结果字段要做只读展示，并通过颜色或背景做轻提示。

### 4. 必填和只读

- 必填字段统一加红色`*`
- 结果字段统一使用只读样式
- 公式说明和提示文字用11-12px弱提示

## 兼容规则

### 1. 端侧优先级

默认：`PC优先，移动兼容查看`

不要为了手机重构成卡片流；在移动端以缩字号、减padding、允许横向阅读为主。

### 2. 响应式断点

建议至少保留以下断点：

- `1180px`
- `900px`
- `720px`

缩放策略：

- 标题从20px降到18px/16px
- 正文从14px降到13px/12px
- padding适度减小
- 明细区和汇总区允许压缩，但不破坏表格主结构

## 生成流程

当用户提供参考文件时，按以下顺序执行：

1. 先判断表单类型：报价、立项、审批、风控、采购、测算等
2. 提取固定字段、长文本字段、明细字段、测算字段
3. 复用本技能的OA样式令牌和布局骨架
4. 按参考文件的字段和业务逻辑填充结构
5. 如有金额/数量/成本关系，补齐前端联动计算
6. 输出为桌面单文件HTML原型

## 必须补齐的内容

如果参考文件没有明确写，但属于正式OA表单常见需求，默认考虑补齐：

- 必填星号
- 日期选择交互
- 汇总/测算逻辑
- 动态子表操作
- 只读结果字段
- 附件上传位
- 移动端兼容样式

## 不要这样做

- 不要做成后台卡片页
- 不要用过重的阴影、渐变、动效
- 不要使用夸张颜色
- 不要大幅偏离浅蓝审批单风格
- 不要为了美观牺牲字段密度和审批阅读效率

## 交付要求

默认交付：

- 1个可直接打开的HTML文件
- 如需要，可同时附带桌面上的参考说明文本

文件命名应贴合场景，例如：

- `OA报价表单原型.html`
- `OA立项表单原型.html`
- `OA风控审批表单.html`
