# 常见故障与修正

这个清单只保留最常见、最容易反复踩的坑。

## 先做的事

1. 先把工作流缩到最小闭环。
2. 先确认文件来源，再确认节点类型。
3. 先保证能导入、能运行，再优化提示词。

## 文件上传相关

- chatflow 场景优先检查 `sys.files`，不要默认用 `start` 新建 file 变量。
- 不要把 `File` 对象直接传给 `template-transform` 或 `llm`。
- 多文件对比时，先拆分成 A/B 两条独立文本链路，再进对比提示词。

## 高发报错

- `Object of type File is not JSON serializable`
  - 原因：把原始文件对象送进了需要序列化的节点。
  - 修法：删掉文件对象参与的模板运算，先走 `document-extractor`。

- `Object of type 'NoneType' has no len()`
  - 原因：模板对空值做了 `len()`。
  - 修法：删掉文件数判断节点，改成更短的直连链路。

- `IfElseNodeData validation error`
  - 原因：`case_id` 不是字符串，或条件操作符写错。
  - 修法：`case_id` 写字符串，只用平台支持的操作符。

## 结构性原则

- 任何“没被调用的节点”都应该优先删除。
- 任何“为了判断而判断”的节点都应该优先删除。
- 如果一条链路要靠 3 个中间节点才能跑通，通常说明结构太长，应该重建。

## 模型切换

- `name` 和 `provider` 必须一起改。
- 改模型前，先确认工作区里真的可用。
- 如果模型只是换成同一家别的型号，先复制一份可运行模板再改。
