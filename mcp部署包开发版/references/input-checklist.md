# 输入检查清单

生成 MCP 部署包前，至少确认以下内容：

## 1. 业务基础信息

- 部署包名称
- 业务名称
- 部署目标：本地 / 内网服务器 / 公网服务器
- 交付对象：开发同事 / 自己本地使用 / 第三方系统

## 2. MCP 工具定义

每个工具至少要明确：

- 工具名
- 工具说明
- 输入参数名
- 参数类型
- 是否必填
- 返回格式：文本 / JSON

## 3. 业务逻辑

- 工具只是固定问答，还是有复杂业务逻辑
- 是否需要解析自然语言
- 是否需要结构化输入
- 是否需要多工具协同
- 是否允许追问或返回缺失字段提示

## 4. 数据源

- 无外部数据源
- Excel / CSV / 本地文件
- 数据库
- HTTP API

如果有数据源，还要明确：

- 连接方式
- 凭据来源
- 是否需要本地兜底文件
- 数据文件是否必须随部署包交付

## 5. 环境变量

至少分成两类：

- 通用部署层：`MCP_*`
- 业务层：`<业务前缀>_*`

## 6. 安全要求

- 是否需要 Bearer Token
- 是否仅限内网
- 是否要配 Nginx / HTTPS

## 8. 未满足条件时

如果以下任一项缺失，不要直接产出正式部署包：

- 工具定义不清
- 关键业务规则不清
- 数据源不清
- 部署目标不清

此时应先补齐输入，再生成部署包。
