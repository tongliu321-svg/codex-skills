# 部署包契约

## 固定层

- `python3.11`
- `.venv`
- `start.sh`
- `MCP_HOST`
- `MCP_PORT`
- `MCP_SSE_PATH`
- `mcp_template/`

## 可变层

- `business/` 下的业务文件
- 业务规则
- 工具名
- 返回字段
- 业务说明文档

## 生成原则

1. 固定层不随业务变化而变化。
2. 可变层必须按当前业务重新生成。
3. 新业务只替换 `business/` 和业务逻辑文件。
4. 部署说明必须明确“哪些可复用，哪些不可复用”。
5. 如果业务依赖 Excel、CSV、数据库导出表、项目清单等结构化文件，业务逻辑必须先完整加载数据，再建立字段关联、索引或映射。
6. 统计类、汇总类、列表类问题必须基于全量数据回答，不得因为默认命中条数限制导致缺项或漏项。
7. 最终回答应输出汇总后的业务结果，不应回放原始文件的逐行扫描过程。
