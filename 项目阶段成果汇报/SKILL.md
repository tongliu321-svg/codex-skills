# 项目阶段成果汇报

## 何时使用

当用户需要把一批项目文件、周报、方案、需求文档、运行数据、会议纪要等，整理成适合给老板、集团领导、管理层汇报的内容时使用本技能。

尤其适用于以下任务：

- 将多个项目文件提炼成阶段性汇报
- 将项目过程材料改写成管理层听得懂的成果汇报
- 输出集团/子公司负责人视角的项目总结
- 从原始文档中提炼项目亮点、价值、赋能对象和关键数据
- 统一多个项目的汇报口径和结构

## 核心原则

### 1. 先看老板关心什么

不要先写项目做了什么，要先回答：

- 赋能了谁
- 解决了什么问题
- 形成了什么价值
- 能不能复制

### 2. 把过程语言翻译成管理语言

常见转换方式：

- `完成POC` -> `完成可行性验证`
- `上线试运行` -> `进入实际使用阶段`
- `搭建工作流/原型` -> `搭建能力底座/完成总体框架`
- `发现问题并调优` -> `形成闭环优化能力`

### 3. 数据只保留最能证明价值的部分

优先提炼：

- 时间节点：启动、上线、运行多久
- 覆盖范围：覆盖部门、基地、场景、模块
- 效率指标：节省多少人天、响应缩短多少
- 经营指标：节约成本、增加收益、异常识别数量
- 质量指标：准确率、丢单率、满意度
- 能力沉淀：规则、流程、模板、平台、底座

不要机械堆数据。每个关键数据后尽量补一句价值判断。

### 4. 单项目按“1个结论 + 2到4个支撑”组织

一个项目里，最容易让老板记住的是：

- 1个总判断
- 2到4个核心数据或亮点
- 1句集团价值判断

不要把所有细节都塞进去。

### 5. 要把单项目成果往集团价值上提

优先使用以下表述方向：

- 样板
- 复制路径
- 共享能力
- 标准化能力
- 底座
- 平台化能力

## 提炼流程

### 第一步：读材料并分类

先把原始文件中的信息分成六类：

1. 项目背景和痛点
2. 已完成事项
3. 上线/运行状态
4. 核心数据
5. 前后对比
6. 下一步计划

### 第二步：找出最强信息

每个项目优先挑这些内容：

- 最硬的时间节点
- 最亮的经营或效率数据
- 最清楚的赋能对象
- 最能体现价值的前后对比
- 最容易上升为集团价值的成果

### 第三步：统一口径

把技术口径、项目口径统一改成管理汇报口径：

- “做了什么”改成“形成了什么能力”
- “完成了哪些模块”改成“支撑了哪些场景”
- “实现了哪些功能”改成“赋能了哪些部门/基地/岗位”

### 第四步：输出汇报结构

默认按以下结构输出单项目内容：

#### 项目背景

控制在2到4句，回答：

- 为什么做
- 原来痛点是什么
- 该项目在集团或单位里的定位是什么

#### 阶段性成果

这是最核心部分，默认写法：

1. 先给结论：已形成什么样板/已赋能谁/已进入什么阶段
2. 再给支撑：上线时间、覆盖范围、核心能力、关键数据
3. 再给对比：原来怎么做，现在怎么做
4. 最后收口：对集团意味着什么

通用句式：

`该项目已……，当前直接赋能……。截至目前，已实现……，覆盖……，形成……。与过去……相比，当前已……。项目的核心价值在于……，为后续……提供了样板/基础/复制路径。`

#### 下一步计划

只回答两个问题：

- 下一步继续做什么
- 最终要走向什么目标

以“深化、推广、复制、平台化”表达为主。

## 常用输出结构

### 结构A：标准管理汇报

适用于大多数场景：

- 项目背景
- 阶段性成果
- 下一步计划

### 结构B：老板速读版

适用于需要更短的汇报：

- 核心结论
- 成果亮点

### 结构C：多项目总述版

适用于多个项目一起汇报：

1. 总体概述
2. 单项目逐项展开

总体概述建议包含：

- 围绕哪几条主线
- 形成了几个样板
- 分别对应什么价值方向

## 数据筛选模板

提炼时优先搜索以下关键词或内容：

- `上线`
- `试运行`
- `运行时间`
- `覆盖`
- `支持`
- `完成`
- `节约`
- `提升`
- `准确率`
- `异常`
- `订单`
- `收益`
- `成本`
- `识别`
- `满意`
- `复制`
- `样板`

## 质量标准

最终内容应满足：

- 能让老板快速知道“这件事值不值得关注”
- 能明确看出“赋能了谁”
- 能看到“最硬的数据”
- 能感受到“前后变化”
- 能上升到“集团价值或组织价值”

如果项目仍在早期阶段，不要硬写成“已经全面落地”。应如实写成：

- 已完成框架搭建
- 已完成高频场景验证
- 已进入试用阶段
- 已具备复制基础

## 不推荐的写法

- 大量罗列功能点，不讲价值
- 只讲过程，不讲结果
- 数据很多，但没有解释意义
- 过度技术化，老板听不懂
- 把仍在规划中的内容写成已经落地

## 一句话方法论

先从材料中找`事实`，再把事实提炼成`数据`，最后把数据上升成`价值`。
